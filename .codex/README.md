# .codex ドキュメントハブ

このディレクトリは、**JIRAで特定の個人へアサインされた課題を、その個人に紐づくプリンタへ自動印刷する仕組み**を設計・実装・運用するためのドキュメントを集約する場所です。

## 目的
- 要件・制約・設計方針を明文化する
- 実装時の判断基準を揃える
- 運用手順と障害対応を標準化する

## ドキュメント一覧
- `docs/requirements.md`: 要件定義（機能/非機能/制約）
- `docs/architecture.md`: 全体アーキテクチャ設計
- `docs/flow.md`: イベント～印刷までの処理フロー
- `docs/operations.md`: 運用・監視・障害対応
- `docs/security.md`: セキュリティ/コンプライアンス観点
- `docs/roadmap.md`: 段階的な導入計画

## 想定スコープ（Phase 1）
- JIRA Webhook を受信
- assignee（担当者）を解決
- 担当者に紐づくプリンタ設定を参照
- 印刷ジョブを生成し、印刷キューへ投入
- 実行結果を監査ログへ保存
