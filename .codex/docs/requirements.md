# 要件定義

## 1. 背景
JIRA課題が個人へアサインされた際、担当者ごとの物理プリンタへ自動で印刷することで、現場での作業開始までのリードタイム短縮を狙う。

## 2. 機能要件
1. **アサインイベント検知**
   - `issue_updated` 等のWebhookから、assignee変更を検知できること。
2. **担当者→プリンタ解決**
   - `accountId`（またはメール）をキーに、対象プリンタIDを解決できること。
3. **印刷データ生成**
   - チケット情報（Key, Summary, Priority, Description, Labels等）から印刷用レイアウトを生成できること。
4. **印刷実行**
   - CUPS / IPP / ベンダーAPI等のいずれかで印刷ジョブを実行できること。
5. **再実行・冪等制御**
   - 同一イベント重複時の二重印刷を防止できること。
6. **監査ログ**
   - 誰の課題を、いつ、どのプリンタに、成功/失敗で出したかを記録すること。

## 3. 非機能要件
- 可用性: 一時的障害時にリトライ可能。
- 性能: Webhook受信からキュー投入まで数秒以内を目標。
- 拡張性: 担当者・拠点増加時の設定追加が容易。
- 保守性: 設定とアプリロジックを分離。

## 4. 制約・前提
- JIRA管理者がWebhook設定可能であること。
- ネットワーク的に印刷先へ到達可能であること。
- 印刷対象情報に個人情報が含まれる可能性を考慮すること。

## 5. 未決事項
- 印刷フォーマット（A4固定/ラベルプリンタ併用）。
- 再印刷トリガ（手動 or 自動）。
- 夜間バッチ抑止や休日ルール。
